# OpenClaw 基础知识

## Agent 智能体核心概念

每个 Agent 都是一个独立的大脑，我们可以把它看作是一个拥有独立身份、特定技能和专属记忆的“数字员工”或者“硅基员工”，它不再只是一个被动的聊天机器人，而是一个能够“理解指令、规划任务、动手执行”的自主实体。

## Agent 工作目录

每个 Agent 都在文件系统中拥有一个完全独立的工作目录，如同一个独立的小世界。它的全部记忆（记忆）、行为准则（配置）和看家本领（技能），都以最通用的纯文本文件形式存放于此。

```txt
~/.openclaw/workspace/
├── .openclaw/                # 存放存储运行时参数
├── skills/                   # 本地技能⽬录
├── AGENTS.md                 # Agent 工作区指南
├── BOOTSTRAP.md              # 启动引导文件
├── HEARTBEAT.md              # 心跳检查配置
├── IDENTITY.md               # 定义 Agent 身份信息
├── SOUL.md                   # 定义 Agent 核心人格定义
├── TOOLS.md                  # 工具配置和本地笔记
└── USER.md                   # 用户信息记录
```

- `TOOLS.md` 文件，用于定义「我有什么工具」，以及如何使用各种工具，以及特定场景下的工具选择策略，内容定义如下：
- `skills/` 目录，是 `Agent` 的 “技能脚本仓库”，每个 Skill 对应一个具体的、可执行的原子功能，Agent 本身不直接实现业务逻辑，而是通过调用 skills/ 下的脚本完成具体工作。
    - Agent 是 “专职员工”，Skill 是员工手上的 “螺丝刀 / 计算器 / 代码生成器”；
    - 一个 Agent 可绑定多个 Skill（如 JavaCodingAgent 绑定「代码生成」「规范检查」「依赖解析」3 个 Skill）；
    - 所有 Skill 需在 TOOLS.md 中注册，Agent 才能调用。


## Agent 智能体和人类对比例子  

| 维度 | Agent 特征 | 人类团队类比 |
| :--- | :--- | :--- |
| **能力专属** | 只聚焦单一类型任务（如「Java 编码」「飞书表格解析」「数据测试」），不做超出能力范围的事 | 开发团队中的「后端工程师」只写代码，「测试工程师」只做测试，分工明确 |
| **工具绑定** | 自带完成任务所需的「工具集」（如 Java 编码 Agent 绑定「代码生成工具」「Maven 依赖检查工具」） | 后端工程师会用 IDE、Maven，测试工程师会用 JUnit、Postman |
| **模型适配** | 绑定最适合该任务的 AI 模型（如代码类 Agent 用 CodeLlama，文档类 Agent 用 Qwen） | 不同岗位用不同工具（设计师用 Figma，程序员用 VS Code） |
| **结果输出** | 按固定格式输出结果（如 JSON/Java 文件 / 测试报告），方便其他 Agent 读取 | 测试工程师输出标准化的测试报告，后端工程师能直接看懂 |
| **可协作** | 支持和其他 Agent 共享数据（通过「记忆」），配合完成复杂任务 | 测试工程师把 bug 报告发给开发工程师，开发修复后反馈给测试 |
